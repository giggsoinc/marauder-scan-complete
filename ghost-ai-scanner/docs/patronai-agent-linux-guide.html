PatronAI Agent for Linux

A lightweight background agent that runs on Linux workstations and servers, detects unauthorised AI tool usage — n8n, Flowise, Langflow, Ollama, LM Studio, MCP servers, git commits with AI code — and reports findings to your company's PatronAI dashboard. Installs in a single bash command. No root required.

1. What the Agent Actually Does

Every 30 minutes the agent wakes up, scans what's installed and running on the Linux machine, compares against the company's approved AI tool list, and uploads a small JSON status file to your company's secure storage. A liveness heartbeat fires every 5 minutes. No continuous network capture, no keylogging, no reading your code.

Why a periodic scan and not just git? Data scientists and MLOps engineers run Ollama, Flowise, and n8n as local services — they never commit code for those. The periodic scan catches local services, Docker workflows, and installed packages that are invisible to git hooks alone.

2. Two Ways to Install

RECOMMENDED Bash installer

  • What ships: setup_agent.sh — self-contained bash script, OTP-protected
  • How you get it: Your IT admin sends a download link from Settings → Deploy Agents; unique to your email, expires in 7 days
  • How you run it:
# Download (link from IT admin email)
curl -fsSL "https://<your-link>" -o setup_agent.sh
chmod +x setup_agent.sh
bash setup_agent.sh
  • What you see: Terminal prompts for OTP → validates → installs agent → sets up cron/systemd timer
  • No sudo: All files written to ~/.patronai/; scheduler registered under your user account

ENTERPRISE Ansible / Chef / Puppet

  • The bash installer is idempotent — run it in your configuration management playbook
  • Pass OTP via environment variable: PATRONAI_OTP=123456 bash setup_agent.sh for non-interactive install
  • Agent state lives entirely in ~/.patronai/ — easy to manage, easy to remove
  • Systemd user unit preferred when available; falls back to crontab automatically
Python 3 and bcrypt are required. The installer checks for both and installs bcrypt via pip if missing. If your machine blocks outbound pip, pre-install bcrypt>=4.0.0 before running the script.

4. The Six Signal Sources

The agent scans six places on every 30-minute cycle. Any signal beyond the authorised list is packaged into a JSON payload and uploaded to your company's PatronAI bucket.

1 · Processes
Running binaries
ps -eo comm — matches n8n, flowise, langflow, ollama, lm-studio, dify, cursor, autogen, crewai process names
2 · Ports
Listening sockets
ss -tlnp (or netstat -tlnp) → 5678 n8n · 3000 flowise · 7860 langflow · 11434 ollama · 1234 lm studio · 8080 generic LLM
3 · Docker
AI containers
docker ps --format → flags ollama/ollama, flowiseai/flowise, dify/*, langflow-ai/* images; also checks Podman if present
4 · MCP servers
Desktop app configs
Reads ~/.config/Claude/claude_desktop_config.json and Cursor MCP settings. Server name + command basename only — env variable values never transmitted.
5 · Packages
pip / conda / npm
pip list --format=json, conda list (if present), npm ls -g — flags langchain, openai, anthropic, llama-index, autogen, crewai, huggingface-hub
6 · Git commits
Pre-commit hook (async)
Installed in all existing repos under $HOME. Fires only on git commit when AI framework patterns are detected in the diff. Diff content hashed, not transmitted plaintext.
What the agent never does
  • No keystrokes, no screen capture, no clipboard, no browser history
  • No continuous network packet capture (that's a network-layer tool's job)
  • No file content upload — only names, paths, and pattern-match results
  • API key values hashed locally — only the hash and file path leave the device
  • MCP config argument values are never transmitted — only the server name and command basename
  • Source code is never read beyond counting AI decorator patterns
  • Does not walk above $HOME; skips .venv, node_modules, __pycache__, .git/objects

5. How the Agent Stays Running on Linux

The installer detects the available scheduler and uses the best option. Systemd user units are preferred — they survive reboots without any root configuration. Cron is the fallback.

PREFERRED systemd user timer

# Check status
systemctl --user status patronai-heartbeat.timer
systemctl --user status patronai-scan.timer

# View logs
journalctl --user -u patronai-heartbeat.service -n 20
journalctl --user -u patronai-scan.service -n 20
  • Survives reboots automatically when linger is enabled
  • Clean start/stop semantics, no zombie processes
  • Unit files written to ~/.config/systemd/user/

FALLBACK crontab

# View installed cron entries
crontab -l | grep patronai

# Expected entries:
*/5 * * * *  ~/.patronai/heartbeat.sh
*/30 * * * * ~/.patronai/scan.sh
  • Used when systemd user sessions are not available
  • Works on all distributions including minimal/server installs
ScriptSchedulePurpose
~/.patronai/heartbeat.shEvery 5 minutesLiveness ping; refreshes presigned URLs from urls_refresh_url; backstops git hook coverage in any new repos cloned since last run
~/.patronai/scan.shEvery 30 minutesFull 6-surface scan — processes, ports, Docker/Podman, MCP configs, packages, Phase 1A signals
The heartbeat refreshes presigned PUT URLs on every 5-minute cycle by fetching a urls.json bundle from the server. This means the agent continues working long after install-time URLs expire — no reinstall needed when presigned URLs rotate.

7. Advanced Scan Signals (Phase 1A)

Additional surfaces scanned on every 30-minute cycle

  • MCP server inventory — reads ~/.config/Claude/claude_desktop_config.json and Cursor, Continue, Cline configs. Sends server name + command basename only. SHA-256 of config file shipped for change detection.
  • Agent workflow inventory — detects ~/.n8n/workflows/, ~/.flowise/, ~/.langflow/flows/. File path (home-redacted to ~), size, and mtime only — workflow content never transmitted.
  • Scheduled-agent inventory — parses crontab -l output and ~/.config/systemd/user/*.service files for AI-related keywords. Schedule expression shipped; command arguments redacted.
  • Tool registration inventory — walks git repos under $HOME (depth 4), counts @tool / Tool() / @function_tool decorators in .py files. Counts only — source code never transmitted. Time-capped at 60 s.
  • Vector DB inventory — looks for Chroma, FAISS, LanceDB, Qdrant, Milvus, DuckDB-vector files in home caches and discovered repos. File name and size only.
  • Shell history scan — one-time scan of ~/.bash_history, ~/.zsh_history for AI-related commands (ollama, langchain, openai, etc.). History is scanned, never uploaded — only the count and command categories are reported.

Privacy gate — secrets never leave the device

Every finding passes through a redactor before upload. Strips:

  • OpenAI / Anthropic keys (sk-..., sk-ant-...)
  • HuggingFace tokens (hf_...)
  • AWS access keys (AKIA...)
  • Google API keys (AIza...)
  • GitHub PATs (ghp_, ghs_, ghu_)
  • Slack tokens (xoxb-)
  • JWTs (three-part dot-separated base64)
  • User home path replaced with ~

If a finding still matches a secret pattern after redaction, the entire finding is dropped.

8. Reinstall and Uninstall

To reinstall (e.g. to pick up new scan modules)

  1. Your IT admin issues a new package from Settings → Deploy Agents
  2. You receive a new download link and a fresh 6-digit OTP
  3. Uninstall the existing agent first: bash ~/.patronai/uninstall.sh
  4. Download and run the new installer
  5. The first scan after install runs in baseline mode — a wider sweep. Subsequent 30-min scans run lighter.

To uninstall completely

# No sudo needed
bash ~/.patronai/uninstall.sh

The uninstaller stops and removes systemd user timers (or crontab entries), strips the pre-commit shim from all git repos under $HOME, restores any backed-up pre-commit hooks, clears git config --global init.templateDir, and removes the ~/.patronai/ directory entirely.

9. Troubleshooting

SymptomLikely causeFix
"OTP invalid"Mis-typed, or package expired (>7 days)Re-type carefully — 6 digits, no spaces. Ask IT admin for a new package if expired.
bcrypt not foundpip blocked by network policyPre-install: pip install --user bcrypt>=4.0.0 or add to your venv before running the installer.
Device shows OFFLINE in fleetTimer not runningCheck: systemctl --user list-timers | grep patronai or crontab -l | grep patronai. Re-run installer if missing.
No findings after installFirst scan still running, or no unauthorised tools presentWait up to 30 min. Force a manual scan: bash ~/.patronai/scan.sh
HTTP 403 in agent.logPresigned URL expired before heartbeat refreshed itWait for next 5-min heartbeat to rotate URLs. If persistent, reinstall.
Git hooks not installed in some reposRepos stored outside $HOME or deeper than depth 4Install manually: cp ~/.patronai/git-template/hooks/pre-commit /path/to/repo/.git/hooks/
Agent not running after reboot (systemd path)linger not enabled for your userEnable: sudo loginctl enable-linger $USER — allows systemd user units to run without an active session.

Self-diagnose

The agent installs a diagnose.sh alongside itself. Run it any time to check timer status, last heartbeat, log tail, and hook coverage:

bash ~/.patronai/diagnose.sh

View the local action log

# Filter for failed uploads
