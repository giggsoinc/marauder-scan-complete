PatronAI Agent for Windows

A lightweight background agent that runs on Windows, detects unauthorised AI tool usage — n8n, Flowise, Langflow, Ollama, LM Studio, Cursor, MCP servers, git commits with AI code — and reports findings to your company's PatronAI dashboard. No admin rights required. One-click installer or one PowerShell command.

1. What the Agent Actually Does

Every 30 minutes the agent wakes up, scans what's installed and running on your Windows machine, compares against the company's approved AI tool list, and uploads a small JSON status file to the company's secure storage. It also sends a liveness heartbeat every 5 minutes. No continuous network capture, no keylogging, no reading your code.

Why a periodic scan and not just git? Many AI tools — n8n, Flowise, Langflow, Ollama — run as local services or Docker containers and never touch git. The periodic scan catches local services, installed packages, and Docker workflows even when no code is committed.

2. Two Ways to Install

RECOMMENDED Windows EXE installer

  • What ships: PatronAI-Agent-Setup.exe — NSIS-built installer, no admin rights needed
  • How you get it: Your IT admin sends a download link from Settings → Deploy Agents; link is unique to you and expires in 7 days
  • What you see: A standard Windows installer wizard — double-click, enter your OTP, click Install
  • Credentials: OTP-protected package — no AWS keys ever touched by the user
  • Auto-starts: Registered as a Task Scheduler task under your user account — runs at login, every 5 min heartbeat, every 30 min scan

ADVANCED PowerShell script

  • What ships: setup_agent.ps1 — same logic, PowerShell 5.1+
  • How you run it: powershell -ExecutionPolicy Bypass -File setup_agent.ps1
  • When to use: Intune / SCCM deployment, automated onboarding, or when the EXE is blocked by AV
  • Same result: Creates %USERPROFILE%\.patronai\, registers Task Scheduler tasks, installs git hooks
Execution policy: The installer needs -ExecutionPolicy Bypass only for the initial install. Once running, the scheduled tasks invoke the PowerShell scripts directly — no persistent policy change is made to your system.

4. The Six Signal Sources

The agent scans six places on every 30-minute cycle. Any signal beyond the authorised list is packaged into the JSON payload and uploaded to your company's PatronAI bucket.

1 · Processes
Running binaries
Get-Process output matched against n8n, flowise, langflow, ollama, lm-studio, dify, cursor, autogen, crewai process names
2 · Ports
Listening sockets
netstat -ano → 5678 n8n · 3000 flowise · 7860 langflow · 11434 ollama · 1234 lm studio · 8080 generic LLM
3 · Docker
AI containers
docker ps --format → flags ollama/ollama, flowiseai/flowise, dify/*, langflow-ai/* images
4 · MCP servers
Desktop app configs
Reads %APPDATA%\Claude\claude_desktop_config.json and Cursor MCP settings. Lists every MCP server name and command — values never transmitted.
5 · Packages
pip / conda / npm
pip list --format=json, conda list, npm ls -g — flags langchain, openai, anthropic, llama-index, autogen, crewai and similar packages
6 · Git commits
Pre-commit hook (async)
Installed in all existing git repos under your user profile. Fires only on git commit. Sends the diff if AI framework patterns detected — content hashed, never plaintext.
What the agent never does
  • No keystrokes, no screen capture, no clipboard access, no browser history
  • No continuous network packet capture
  • No file content upload — only names, paths, and pattern matches
  • API key values are hashed locally — only the hash and file path leave the device
  • MCP config values (env vars, arguments) are never transmitted — only the server name and command basename
  • Source code is never read beyond counting AI decorator patterns (@tool, Tool())

7. Advanced Scan Signals (Phase 1A)

Additional surfaces scanned on every 30-minute cycle

  • MCP server inventory — reads Claude Desktop (%APPDATA%\Claude\claude_desktop_config.json), Cursor, Continue, Cline configs. Sends server name + command basename only. Config file SHA-256 shipped for change detection.
  • Agent workflow inventory — detects n8n data directories, Flowise flows, Langflow YAML files. File path, size, and mtime shipped — workflow contents never leave the machine.
  • Scheduled-agent inventory — queries Task Scheduler for jobs with AI-related keywords in the command line. Schedule expression shipped; command arguments redacted.
  • Tool registration inventory — walks git repos under %USERPROFILE%, counts @tool / Tool() / @function_tool decorators in .py files. Counts only — source code never transmitted.
  • Vector DB inventory — looks for Chroma, FAISS, LanceDB, Qdrant, Milvus, DuckDB-vector files. File name and size only — content never read.

Privacy gate — secrets never leave the device

Every finding passes through a redactor before upload. Strips:

  • OpenAI / Anthropic keys (sk-..., sk-ant-...)
  • HuggingFace tokens (hf_...)
  • AWS access keys (AKIA...)
  • Google API keys (AIza...)
  • GitHub PATs (ghp_, ghs_, ghu_)
  • Slack tokens (xoxb-)
  • JWTs (three-part dot-separated base64)
  • User home path replaced with ~

If a finding still matches a secret pattern after redaction, the entire finding is dropped. Better to lose visibility than leak credentials.

8. Reinstall and Uninstall

To reinstall (e.g. to pick up new scan modules)

  1. Your IT admin issues a new package from Settings → Deploy Agents in the dashboard
  2. You receive a new download link and a fresh 6-digit OTP by email
  3. Run the existing uninstaller first: powershell -ExecutionPolicy Bypass -File %USERPROFILE%\.patronai\uninstall.ps1
  4. Download and run the new PatronAI-Agent-Setup.exe
  5. Enter the new OTP when prompted
  6. The first scan after install runs in baseline mode — a wider sweep. Subsequent scans are lighter.

To uninstall completely

# Run in PowerShell (no admin needed)
powershell -ExecutionPolicy Bypass -File "$env:USERPROFILE\.patronai\uninstall.ps1"

The uninstaller removes both Task Scheduler tasks, strips the pre-commit hook from all git repos under your profile, clears git config --global init.templateDir if it was set by PatronAI, and deletes the .patronai\ directory.

Your code and git history are completely untouched.

9. Troubleshooting

SymptomLikely causeFix
SmartScreen blocks the EXEEXE downloaded from email / untrusted zoneClick More info → Run anyway. The EXE is signed. If your AV quarantines it, add an exclusion for PatronAI-Agent-Setup.exe or use the PowerShell path instead.
"OTP invalid" at installOTP mis-typed, or package expiredCheck the email carefully — 6 digits, no spaces. If the package is older than 7 days, ask your IT admin to issue a new one.
Device shows OFFLINE in fleetTask Scheduler tasks not runningRun: Get-ScheduledTask -TaskName "PatronAI*" | Select TaskName,State — both should show Ready. If Disabled, re-run the installer.
No findings after installFirst scan still running, or no unauthorised tools presentWait up to 30 min for the first scan. Run manually: powershell -File "%USERPROFILE%\.patronai\scan.ps1"
Agent log shows HTTP 403Presigned URL expiredHeartbeat refreshes URLs on each cycle — wait for the next 5-min heartbeat. If 403 persists, reinstall.
Git hooks not installingRepos not under %USERPROFILE% (depth limit 4)The installer walks 4 directory levels from your home. Repos stored on other drives need manual hook install: copy "%USERPROFILE%\.patronai\git-template\hooks\pre-commit" path\to\repo\.git\hooks\

Self-diagnose script

A diagnose.ps1 is installed alongside the agent. Run it any time to check task status, last heartbeat time, log tail, and hook coverage:

powershell -ExecutionPolicy Bypass -File "$env:USERPROFILE\.patronai\diagnose.ps1"
