EC2 · run once

EC2 · run once

5.5Add custom rules — Provider Lists
BROWSER · Settings → Provider Lists
Two deny lists ship with the product (network domains and committed-code patterns). Both are read-only in the UI — Giggso maintains them via product releases. Customers extend them via two custom lists that survive Docker rebuilds and are merged at scan time. On a collision the custom row wins, letting you locally tighten severity or add notes.
  1. Click ⚙ Settings → Provider Lists
  2. Expand Show baseline rules if you want to see what already ships
  3. Scroll to Network denylist — Custom additions. Add a row by typing or pasting
  4. Forgive-input is built in — paste https://Flowiseai.com/login and on save it normalises to flowiseai.com. Smart quotes, paths, mixed case, trailing dots, zero-width characters are stripped automatically
  5. Click Save
ℹ Inline error feedback. If a row fails validation (port out of range, pattern too broad like *.com, malformed glob), the save aborts and a red table lists the bad rows with reasons. Fix and re-save.
⚠ Conflict gate. If your new deny rule would already be suppressed by an entry in your Allow list, a yellow banner appears. Click Save anyway (override) if intentional — the override is recorded in the audit log.
The same flow exists for the Code denylist — Custom additions (Marauder Scan layer). Patterns there are matched against committed code diffs uploaded by hook agents.
A Last 5 changes expander at the bottom of the tab shows recent edits — who, when, field, before → after. Reads from the existing audit prefix.
5.5.1Bulk-upload a CSV
BROWSER · Settings → Provider Lists
Got a long list of domains in a spreadsheet? Skip the manual entry — drop the CSV in. The same forgive-input pipeline runs at upload time, so paste-quality issues from Excel, Word, or Slack get cleaned automatically.
  1. In the Custom additions section, expand 📥 Bulk import from CSV
  2. Drop your CSV. The headers must match the editor columns (e.g. name,category,domain,port,severity,notes for the network deny list)
  3. The system instantly shows ✓ N valid · ⚠ M skipped. Schemes, paths, smart quotes, zero-width chars, BOM markers, mixed case, and trailing dots are all normalised silently — admin sees the cleaned form
  4. If any rows were skipped, expand Show M skipped rows to see the reason per line. Click Download issues CSV to fix them offline and re-upload
  5. Click Load N valid rows into editor. Rows append to the editor (deduped on key columns; imported wins on collision)
  6. Edit further if needed, then click Save
🔎 Find tool. The text input above the editor highlights matching rows without changing the editor. Use it to locate flowiseai.com in a list of 200 rules without scrolling.
🗑 Clear editor. The Clear button next to Save empties the editor in your session. Click Save with an empty editor to wipe the S3 file (recorded in the audit log). Useful when re-uploading a fresh source-of-truth CSV.
5.5.2Triage discovered AI tools
BROWSER · Settings → Provider Lists
Manual deny-list maintenance doesn't scale — every new AI tool launch becomes another row to add. The bottom of Provider Lists shows a Discovered AI tools — review queue section that turns the matcher's UNKNOWN verdicts into actionable triage.
  1. Scroll to the bottom of Provider Lists
  2. The table shows domains the matcher saw in the last 7 days that didn't match either your allow list or the deny baseline — ranked by event count, with sample device and last-seen timestamp
  3. Use the multiselect to pick one or more rows
  4. Click Promote to deny — domains are appended to your unauthorized_custom.csv with severity HIGH and notes "Auto-promoted from review queue". Audit-logged. Within one scan cycle the new rules fire
  5. Click Dismiss for false positives — domain hides from the queue (persisted to config/discovered_dismissed.txt)
ℹ Why this matters. The matcher's "never silently pass" contract means every unrecognised destination gets logged as UNKNOWN. The review queue turns that log into a self-discovering deny-list — your users find the new AI tools for you, instead of Giggso reading TechCrunch.
🔄 Coverage update propagation. Endpoint-scan changes (browsers, IDE plugins, containers, shell history) apply only to freshly issued installers — existing fleet members keep their old scan logic until re-issued. Coordinate fleet refreshes via MDM rather than expecting per-update auto-rerenders.
ℹ Strict-mode boot. If the merged deny list falls below STRICT_MIN_RULES (default 50), a red banner appears at the top of Provider Lists and a CRITICAL degraded_ruleset finding is fired to SNS/Trinity. The dashboard stays up — fix it from the UI.
Phase 6 · Per developer
6
Deploy hook agents to developers
6.1Generate and send an installer (Admin)
BROWSER · Settings → Deploy Agents
  1. Go to Settings → Deploy Agents tab
  2. Fill in the developer's Name, Email, and Platform (Mac / Linux / Windows)
  3. Optionally enter Authorised tools — one domain per line (e.g. canva.com, figma.com). These are suppressed from scan findings for this user only. Editable later without reinstalling.
  4. Click Generate & Send
  5. Developer receives an email with a 48-hour download link and a 6-digit OTP
  6. DMG (Mac) and EXE (Windows) are built automatically on EC2 — download links appear in the table within 30 seconds
  7. Status column shows PENDING → INSTALLED once they run it
  8. To edit the whitelist later: click Whitelist on the table row → edit → Save. Agent picks up changes within 30 min, no reinstall needed.
6.2Developer installs on Mac
DEVELOPER MACHINE · Terminal
Paste the link from the email and run:
curl -fsSL "<link-from-email>" | bash
OTP promptEnter the 6-digit code from the email
Installs to~/.patronai/config.json
Heartbeatlaunchd plist registered — fires every 5 min
Git hooksInstalled in all repos found under ~/, /workspace, /srv

6.3Developer installs on Linux
DEVELOPER MACHINE · Terminal
Same installer as Mac — the script detects the OS and switches to crontab automatically:
curl -fsSL "<link-from-email>" | bash
OTP promptEnter the 6-digit code from the email
Installs to~/.patronai/config.json
Heartbeatcrontab entry added — */5 * * * *
Git hooksInstalled in all repos found under ~/, /workspace, /srv
ℹ Requires curl and python3. bcrypt is installed automatically if missing.
6.4Developer installs on Windows
DEVELOPER MACHINE · PowerShell
Step 1 — Download the installer. Paste this in PowerShell (no Admin needed for download):
Invoke-WebRequest -Uri "<link-from-email>" -OutFile "$env:USERPROFILE\setup_agent.ps1"
Step 2 — Run the installer. Still in the same PowerShell window:
powershell -ExecutionPolicy Bypass -File "$env:USERPROFILE\setup_agent.ps1"
OTP promptEnter the 6-digit code from the email
Installs to%USERPROFILE%\.patronai\config.json
HeartbeatWindows Task Scheduler task registered — PatronAI-Heartbeat, every 5 min
Git hooksInstalled in all repos found under %USERPROFILE% (max depth 4)
⚠ Python 3 must be installed and on the PATH before running the installer. Get it from python.org — tick "Add Python to PATH" during install. bcrypt is installed automatically.
Step 3 — Verify. Check installation and Task Scheduler entry:
# Confirm config was written
Get-Content "$env:USERPROFILE\.patronai\config.json"

# Confirm heartbeat task is registered
Get-ScheduledTask -TaskName "PatronAI-Heartbeat"
ℹ The pre-commit hook is a .ps1 script. Git calls it via a thin pre-commit shell shim that Git for Windows understands. Ensure Git for Windows is installed.
6.5Download the one-click installers (DMG & EXE — built automatically on EC2)
BROWSER · Deploy Agents tab
PatronAI builds both a macOS .dmg and a Windows .exe on the EC2 server at package generation time — no macOS or Windows machine required. The Deploy Agents table shows Download DMG and Download EXE links once the build completes (usually within 30 seconds of clicking Generate & Send).
Mac developer — receives the link via email, or admin copies the DMG download link from the table.
Double-click the DMG to mount it → double-click PatronAI-Agent-Alice.command → Terminal opens and runs the installer.
Windows developer — download the EXE, double-click → silent install → PowerShell script runs automatically.
Both artifacts are stored at:
s3://<bucket>/config/HOOK_AGENTS/<token>/PatronAI-Agent-<firstname>.dmg
s3://<bucket>/config/HOOK_AGENTS/<token>/PatronAI-Agent-<firstname>.exe
