User Guide · macOS Edge Agent · v1.1

PatronAI Agent for Mac

A background monitor that runs on your Mac, detects unauthorised AI tool usage — n8n, Flowise, Langflow, sim.ai, Ollama, Cursor, LM Studio, MCP servers, git commits with AI code — and reports findings to your company's PatronAI bucket. No command line. No sudo. One-click install.

1. What the Agent Actually Does

Every 5 minutes the agent wakes up, looks at what's running on your Mac, compares against the company's approved AI tool list, and writes a small JSON status file to the company S3 bucket. That's it. No continuous network capture, no keylogging, no inference on your device.

Why a heartbeat and not just git? Marketing and ops teams run n8n, Flowise, sim.ai and Langflow as local services — they never touch git. Git hook alone would miss every one of them. The heartbeat catches local services, desktop apps and Docker workflows even when no code is ever committed.

2. Two Ways to Get the Agent

PRODUCTION Signed DMG via Jamf / Wiki

  • What ships: PatronAI-Agent-1.1.dmg (notarised, signed with Giggso Developer ID)
  • How user gets it: pushed automatically by Jamf, or downloaded from the company wiki after SSO login
  • What user sees: a normal Mac installer — no Terminal, no sudo
  • Credentials baked in: the DMG carries a pre-provisioned scoped IAM role, one per customer, issued by Giggso
  • Script we write: agent/install/macos/build-dmg.sh (builds the DMG) plus jamf.sh (Jamf policy wrapper)

POC / DEBUG curl one-liner for devs

  • What ships: debugger_jamf.sh — a self-contained bash script
  • How user gets it: curl -fsSL https://…/debugger_jamf.sh | bash
  • What user sees: Terminal output; this is for engineers smoke-testing the pipeline, not for end users
  • Credentials: prompts for AWS access key + secret, or reads from existing ~/.aws/credentials
  • Script we write: agent/install/macos/debugger_jamf.sh
POC only. debugger_jamf.sh is not a product deliverable — it exists so you and the engineering team can validate the data path into S3 without waiting on the DMG build. Never hand this version to an end user.

4. The Six Signal Sources

The agent scans six places on every heartbeat. Any signal beyond the authorised list is packaged into the JSON payload and shipped to S3.

1 · Processes
Running binaries
ps -axo comm → match against n8n, flowise, langflow, ollama, lm-studio, dify, autogen, crewai binaries
2 · Ports
Listening sockets
lsof -nP -iTCP -sTCP:LISTEN → 5678 n8n · 3000 flowise · 7860 langflow · 11434 ollama · 1234 lm studio
3 · Docker
AI containers
docker ps --format → flag ollama/ollama, flowiseai/flowise, dify/*, langflow-ai/* images
4 · MCP servers
Desktop app configs
Reads ~/Library/Application Support/Claude/claude_desktop_config.json and Cursor MCP settings — lists every MCP server wired up
5 · API keys
Env and dotfiles
Grep for OPENAI_API_KEY, sk-ant-, hf_ prefixes in ~/.env, ~/.zshrc, project .env files. Key values are hashed, never transmitted.
6 · Git commits
Pre-commit hook (async)
Separate path — fires only when user runs git commit. Ships the diff if AI framework patterns are present.
What the agent never does
  • No keystrokes, no screen capture, no browser history
  • No continuous network packet capture (that's Packetbeat's job, on the network side)
  • No file content upload — only names, paths and pattern matches
  • API key values are hashed locally — only the hash and the file location leave the device

9. Approval Checklist Before I Write Code

  • Heartbeat interval of 5 minutes is acceptable (not 60 s, not 15 min)
  • Six signal sources are the right list (processes, ports, Docker, MCP configs, API keys, git)
  • Distribution = signed notarised DMG (prod) + debugger_jamf.sh (POC only)
  • No admin password during install — Login Item via SMAppService is fine
  • API key values hashed locally, only hashes leave the device
  • Offline queue caps at 24 h — after that, oldest heartbeats drop
  • S3 path ocsf/agent/heartbeat/YYYY/MM/DD/ alongside existing ocsf/agent/git-diffs/
  • Same JSON schema normaliser (normalizer/agent.py) handles both heartbeat and git-diff event types

10. Phase 1A — what your re-installed agent does that the old one didn't

7 new scan fragments run on every cycle

  • MCP server inventory — reads Claude Desktop / Cursor / Continue / Cline JSON configs. Sends server name + command basename + arg flags only (NEVER values, NEVER environment variables values). SHA-256 of the config file is uploaded for change detection.
  • Agent workflow inventory — finds ~/.n8n/workflows/*.json, ~/.flowise/, ~/.langflow/flows/*.yaml on disk. File path redacted to ~/...; bytes + mtime shipped. Workflow contents NEVER leave your laptop.
  • Scheduled-agent inventory — parses crontab -l output and ~/Library/LaunchAgents/*.plist for AI keywords. Schedule expression shipped; command line redacted via the secret regex first.
  • Tool registration inventory — auto-discovers your git repos (walks $HOME for .git/ dirs), greps *.py files for @tool / @function_tool / Tool() decorators. Counts only — source code never ships.
  • Vector DB inventory — looks for Chroma / FAISS / LanceDB / Qdrant / Milvus / DuckDB-vector files in home caches AND inside discovered repos. File NAME and SIZE shipped; content NEVER read.

Privacy gate — secrets never leak

Every Phase 1A finding goes through a shared redactor before upload. Strips:

  • OpenAI / Anthropic API keys (sk-..., sk-ant-...)
  • HuggingFace tokens (hf_...)
  • AWS access keys (AKIA...)
  • Google API keys (AIza...)
  • GitHub PATs (ghp_, gho_, ghu_, ghs_)
  • Slack tokens (xoxb- family)
  • JWTs (three-part dot-separated base64)
  • User home dir replaced with ~

If a finding still contains a secret-pattern match after redaction, the entire finding is dropped. We never ship a half-redacted payload — better to lose visibility than to leak credentials.

Re-install procedure

  1. Open the dashboard → Settings → Deploy Agents
  2. Click Re-issue for your email. Download the new .pkg / .sh installer.
  3. On your Mac: bash ~/.patronai/uninstall.sh (removes the old agent)
  4. Run the new installer. It will create ~/.patronai/first_run.flag at install time.
  5. Wait for the next scan cycle (every 30 min). The first scan runs in baseline mode — wider walk, broader file-type matching. The flag clears after a successful payload print, so subsequent recurring scans run lighter.
  6. Within ~5–10 minutes of the first scan, the dashboard's AI Inventory tab populates with your real findings.

What the agent does NOT do

  • Does not upload your MCP config files or workflow JSON contents — only metadata + a hash for change detection
  • Does not ship environment variable VALUES — only the keys (so you know OPENAI_API_KEY is set without revealing the key itself)
  • Does not read source code beyond decorator counts — your code never leaves the laptop
  • Does not recurse into node_modules, .venv, Library/, Applications/, or .Trash
  • Does not walk above $HOME (system files are out of scope)
  • Time-capped at 60 s for repo discovery + 30 s for tool grep — never runs away
