Privacy gate — every Phase 1A finding goes through `_safe_finding()` (redactor) BEFORE upload. API keys, JWTs, AWS access keys, GitHub PATs, Slack tokens are stripped; user-home paths replaced with `~`. Findings that still contain a secret regex match after redaction are dropped entirely — better to lose visibility than to leak.
Repo scope — Phase 1A scanners stay inside discovered git repos (auto-walked from `$HOME` looking for `.git/`). They do NOT trawl the home dir; they do NOT read system dirs (`Library`, `Applications`, `.Trash`). Depth-capped at 6 levels; time-capped at 60 seconds.
