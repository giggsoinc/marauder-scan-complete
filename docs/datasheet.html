1. Product Overview

PatronAI helps security, platform, and AI-governance teams discover unknown AI endpoints, detect abandoned AI assets, and monitor model usage across applications, developer environments, and cloud services. It ingests network telemetry (VPC Flow Logs · Packetbeat · Zeek), normalises events to the OCSF schema, and matches against a curated deny-all list of 70+ AI providers.

A code-scanning layer (Marauder Scan) detects AI framework imports, MCP server registrations, and hardcoded API keys at git-commit time. An on-prem AI chat assistant (LFM2.5-1.2B-Thinking via llama.cpp) answers natural-language questions over your findings — data never leaves your environment.

Single Container

One Docker image. Scanner + Streamlit UI + Grafana + on-prem LLM. Deploys on a t3.large EC2 in under 10 minutes.

Multi-Cloud Ready

S3-native today; GCS, Azure Blob, OCI Object Storage on the v2.0 roadmap. Storage backend is pluggable.

Zero Data Egress

All telemetry stays in your AWS account. The on-prem LLM runs locally — no third-party AI API calls required.

OCSF Native

Every event normalised to OCSF 1.x. SIEM-ready out of the box.

3. Architecture

Three layers feed a single S3 bucket; one Docker container reads from it. No external control plane.

LayerSourcesOutput
EdgeGit pre-commit hook · Endpoint scan agent · Heartbeat (5 min)OCSF events → s3://<bucket>/ocsf/agent/scans/{token}/latest.json
NetworkVPC Flow Logs · Packetbeat · ZeekOCSF events → s3://<bucket>/ocsf/network/...
PlatformScanner engine + Streamlit UI + Grafana + llama-server + nginxFindings → findings/YYYY/MM/DD/{severity}.jsonl · Rollups → users/.../rollup/... · Alerts → SNS / Slack / Teams

7. Security & Compliance

  • Data residency: All findings, rollups, and chat history live in your S3 bucket. No vendor cloud plane.
  • Encryption: SSE-S3 / SSE-KMS supported. SigV4 enforced on all S3 calls.
  • Auth: Email allow-list today; SSO (OIDC / SAML + IdP MFA) on v1.1.
  • Network: Internal LLM bound to 127.0.0.1; reverse proxy via nginx; ALB + ACM TLS recommended in front (v1.1).
  • Audit trail: Every finding immutable in S3; SHA-256-hashed compliance report.
  • Secret hygiene: Grafana admin password fail-loud at boot if unset; no default credentials shipped.
  • Container hardening: Per-service log rotation; healthchecks on all services; non-root container processes where supported.
  • Compliance mappings: NIST CSF and ISO 27001 control mapping in PDF report (v1.2).
